# 部署与运维文档

## 1. 目的
指导实施人员与运维人员进行系统的初始化安装、上线部署、日常监控及灾难恢复。

## 2. 部署指南

### 2.1 环境配置要求
- **OS**: Linux (CentOS 7+ / Ubuntu 20.04+) 或 macOS
- **运行环境**: Python 3.13+, Node.js 18+ (用于前端构建)
- **数据库**: SQLite 3 (内置)
- **Nginx**: 1.18+ (用于反向代理与 WebSocket 支持)

### 2.2 安装与启动步骤
由于项目提供了 `run.sh` 脚本，支持一键启动：
1. **授权脚本**：`chmod +x run.sh`
2. **执行启动**：`./run.sh`
   - *脚本内幕*：会自动创建 Python 虚拟环境、安装 `requirements.txt`、执行前端 Vite 构建（如有）、并启动 Uvicorn 服务器。
3. **访问地址**：前端界面 `http://localhost:8000/`，接口文档 `http://localhost:8000/docs`。

### 2.3 运维管理
- **日志查看**：FastAPI 默认将日志输出至控制台，生产环境建议在启动脚本中配置重定向：`uvicorn main:app > /var/log/smart_table.log 2>&1 &`
- **健康检查**：配置系统定时访问 `GET /api/health`（需开发添加），验证数据库连接状态。
- **备份恢复策略**：
  - 由于当前采用 SQLite (`sql_app.db` / `smart_table.db`)，备份策略为：每天凌晨通过 `cron` 定时拷贝 `.db` 文件至备份目录，保留 30 天。
  - 恢复时，停止服务，替换 `.db` 文件，重启即可。

### 2.4 Nginx 配置示例 (WebSocket 支持)
生产环境使用 Nginx 反向代理时，需显式配置 WebSocket 升级头：

```nginx
server {
    listen 80;
    server_name smart-table.school.edu;

    location / {
        root /var/www/smart_table/dist;
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # WebSocket 专属配置
    location /api/ws/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

## 4. 环境变量说明 (`.env`)
项目根目录下需创建 `.env` 文件（请参考 `.env.example`）：
```ini
# Database
DATABASE_URL=sqlite+aiosqlite:///./smart_table.db
# DATABASE_URL=postgresql+asyncpg://user:pass@localhost/dbname

# Security
SECRET_KEY=change_this_to_a_secure_random_string_32_chars
ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=1440

# App
DEBUG=True
```
