- **渲染逻辑 (`DynamicSurvey.vue`)**:
  1. 遍历 `survey_schema` 数组。
  2. 使用 Vue 的 `<component :is="getComponentType(item.type)">` 进行动态组件渲染。
  3. `getComponentType` 函数映射关系：`text` -> `AInput`，`number` -> `AInputNumber`，`table` -> 自定义 `DynamicTable.vue`。
  4. 使用一个响应式对象 `formData` 来绑定所有动态生成的字段，键名为 `item.key`。

#### 2.2.1 SQLAlchemy Models (`models.py`)
针对 SQLite 不原生支持 JSON 类型的问题，需自定义 `JSON` 类型装饰器。

```python
from sqlalchemy import Column, Integer, String, DateTime, ForeignKey, Text, TypeDecorator
from sqlalchemy.orm import relationship
from sqlalchemy.ext.declarative import declarative_base
import json
from datetime import datetime

Base = declarative_base()

# SQLite 兼容的 JSON 类型
class JSONType(TypeDecorator):
    impl = Text
    cache_ok = True

    def process_bind_param(self, value, dialect):
        if value is None: return None
        return json.dumps(value, ensure_ascii=False)

    def process_result_value(self, value, dialect):
        if value is None: return None
        return json.loads(value)

class User(Base):
    __tablename__ = "users"
    id = Column(String(50), primary_key=True)  # 工号
    serial_no = Column(String(50), nullable=False)
    name = Column(String(100), nullable=False)
    subject = Column(String(50), nullable=False)
    role = Column(String(20), default="teacher") # admin/teacher
    password_hash = Column(String(128), nullable=False) # 新增：密码哈希
    created_at = Column(DateTime, default=datetime.now)

class Survey(Base):
    __tablename__ = "surveys"
    id = Column(Integer, primary_key=True, index=True)
    title = Column(String(255), nullable=False)
    status = Column(String(50), default="draft")
    survey_schema = Column(JSONType, nullable=False) # 问卷结构
    deadline = Column(DateTime, nullable=True)
    created_at = Column(DateTime, default=datetime.now)
    
    responses = relationship("SurveyResponse", back_populates="survey", cascade="all, delete")

class SurveyResponse(Base):
    __tablename__ = "survey_responses"
    id = Column(Integer, primary_key=True, index=True)
    survey_id = Column(Integer, ForeignKey("surveys.id"))
    filler_id = Column(String(50), ForeignKey("users.id"))
    filler_name = Column(String(100), nullable=False)
    dynamic_data = Column(JSONType, nullable=False) # 答卷数据
    created_at = Column(DateTime, default=datetime.now)
    
    survey = relationship("Survey", back_populates="responses")
```

### 2.3 状态机流转设计
为了保证数据的准确性，系统对问卷实施严格的状态管理：
- **草稿 (draft)**：允许修改 schema，不允许提交答卷。
- **已下发 (published)**：不允许修改 schema，允许教师提交答卷。
- **已结束 (closed)**：不允许修改 schema，不允许提交答卷。
后端在执行编辑、下发、撤回、提交答卷等操作时，必须首先检查当前问卷状态是否合法。

### 2.4 实时数据广播设计 (WebSocket)
- **连接管理**: 使用 FastAPI 的 `WebSocketEndpoint`，维护一个 `ConnectionManager` 单例。
- **房间机制**: 基于 `survey_id` 进行频道隔离。客户端连接时需订阅特定 `survey_id`。
  - URI: `/api/ws/surveys/{survey_id}`
- **前端 WebSocket 封装 (`utils/websocket.ts`)**:
  - 实现自动重连机制（指数退避算法）。
  - 实现心跳检测 (Ping/Pong) 防止连接假死。

### 2.5 安全与认证实现
- **JWT 认证**:
  - 算法: HS256
  - Token 有效期: 24小时 (Access Token)
  - 载荷 (Payload): `{ sub: user_id, role: user_role, name: user_name }`
- **密码处理**:
  - 使用 `passlib[bcrypt]` 进行哈希存储与验证。
  - 初始管理员账号由系统初始化脚本生成（见部署文档）。

## 3. 核心 Pydantic Schemas (`schemas.py`)
确保前后端数据交互严格符合契约。

```python
from pydantic import BaseModel
from typing import List, Dict, Any, Optional
from datetime import datetime

class SurveyCreate(BaseModel):
    title: str
    survey_schema: Dict[str, Any] # 或 List[Dict]
    deadline: Optional[datetime] = None

class SurveyResponseCreate(BaseModel):
    dynamic_data: Dict[str, Any]

class UserLogin(BaseModel):
    username: str
    password: str

class Token(BaseModel):
    access_token: str
    token_type: str
    user_info: Dict[str, Any]
```
